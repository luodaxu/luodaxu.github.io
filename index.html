React的四个概念

React主要由四个概念构成，即Virtual DOM，组件化，JSX语法，单项数据流.

Virtual DOM

引入虚拟DOM的意义
  • 虚拟DOM是React的基石，是React支柱。
  • 之所以引入虚拟节点，主要是性能考虑，Web应用和普通网站不一样，一个web应用存在单页面内存在大量的DOM操作，而DOM操作很慢。
  • 在React中，应用程序在虚拟DOM上操作，这让React有了优化的机会，通过diff算法来尽可能小的更改DOM。
  • 除了性能，引入虚拟DOM有有更重要的意义是提供了一种一致的开发方式来开发服务端应用，web应用和手机端应用。


有了虚拟DOM这一层，所以通过不同的渲染器，就可以将内容渲染到不同的平台上，相当好的思路！
至于Virtual DOM速度快的原因另外章节将讲解。

React组件化

组件化概念
  1. 虚拟DOM不仅带来了简答的UI开发逻辑，也带来了组件化开发的思想。所谓组件，即封装起来具有独立功能的部件。React推荐组件的方式重新思考UI构成，将UI上每一个功能相对独立的模块定义成组件，然后将小的组件通过组合或者嵌套的方式构成大的组件， 最终完成整体UI的构建。
  2. 如果说MVC的思想做到了数据－视图－控制器的分离，那么组件化的思想方式则是带来了UI功能模块之间的分离。
  3. 开发者从技术的角度对UI进行拆分，实现松耦合。

对于React而言，则完全是一个新的思路，开发者从功能的角度出发，将UI分成不同的组件，每个组件都独立封装。
在React中，你按照界面模块自然划分的方式来组织和编写你的代码，对于评论界面而言，整个UI是一个通过小组件构成的大组件，每个组件只关心自己部分的逻辑，彼此独立。

  1. 组件特征：React认为组件应有如下特性：

    1. 可组合（Composeable）：一个组件易于和其它组件一起使用，或者嵌套在另一个组件内部。如果一个组件内部创建了另一个组件，那么说父组件拥有（own）它创建的子组件，通过这个特性，一个复杂的UI可以拆分成多个简单的UI组件；
    2. 可重用（Reusable）：每个组件都是具有独立功能的，它可以被使用在多个UI场景；
    3. 可维护（Maintainable）：每个小的组件仅仅包含自身的逻辑，更容易被理解和维护；
    4. 可测试（Testable）：因为每个组件都是独立的，那么对于各个组件分别测试显然要比对于整个UI进行测试容易的多。

  2. 组件定义；在React中定义一个组件也是相当的容易，组件就是一个 实现预定义接口的JavaScript类；

JSX语法

在用React写组件的时候，通常会用到JSX语法，粗看上去，像是在Javascript代码里直接写起了XML标签，实质上这只是一个语法糖，每一个 XML标签都会被JSX转换工具转换成纯Javascript代码，当然你想直接使用纯Javascript代码写也是可以的，只是利用JSX，组件的结 构和组件之间的关系看上去更加清晰

Data Flow（单向数据流）

flux当中重要的是大量的 Model 归到了 Store, View 也统一了,从而得到了所谓单向的数据流, 就是 Model 和 View 之间关系非常清晰了。这样需要人为管理的状态就一下少了很多, 结果体现在开发应用的效率当中。

  1. 一个Flux应用中主要包含四个部分：

  2. 单项数据流
    Flux核心单项数据流： Action -> dispatcher -> Store -> Viewflux

  3. 数据流流程

  • 首先要有 action，通过定义一些 action creator 方法根据需要创建 Action 提供给 dispatcher
  • View 层通过用户交互（比如 onClick）会触发 Action
  • Dispatcher 会分发触发的 Action 给所有注册的 Store 的回调函数
  • Store 回调函数根据接收的 Action 更新自身数据之后会触发一个 change 事件通知 View 数据更改了
  • View 会监听这个 change 事件，拿到对应的新数据并调用 setState 更新组件 UI所有的状态都由 Store 来维护，通过 Action 传递数据，构成了如上所述的单向数据流循环，所以应用中的各部分分工就相当明确，高度解耦了。

这种单向数据流使得整个系统都是透明可预测的

hexo常用命令

hexo

npm install hexo -g #安装
npm update hexo -g #升级
hexo init #初始化

简写

hexo n "我的博客" == hexo new "我的博客" #新建文章
hexo p == hexo publish
hexo g == hexo generate #生成
hexo s == hexo server #启动服务预览
hexo d == hexo deploy #部署

服务器

hexo server #hexo会监视文件变动并自动更新，无须重启服务器
hexo server -s #静态模式
hexo server -p 5000 #更改端口号
hexo server -i 192.168.1.1 #自定义 IP
hexo clean #清除缓存 网页正常情况下可以忽略此条命令
hexo g #生成静态网页
hexo d #开始部署

监视文件变动

hexo generate #使用hexo生成静态文件快速而且简单
hexo generate --watch #监视文件变动

完成后部署

两个命令的作用是相同的
hexo generate --deploy == hexo generate -d
hexo deploy --generate == hexo deploy -g

hexo server #开启预览访问端口（默认端口4000, ‘ctrl+c’终止server）
hexo deploy #将.deploy目录部署到GitHub

hexo new [layout] <title>
hexo new photo "My Gallery"
hexo new "Hello World" --lang tw

css3-placeholder

下面讲解如何修改输入框placeholder文字默认颜色 -webkit-input-placeholder


html5为input添加了原生的占位符属性placeholder，高级浏览器都支持这个属性，例如
<input type="text" value="" placeholder="your name..." />

默认的placeholder字体颜色是浅灰色的，如果想改变这个默认颜色，解决办法：

:-moz-placeholder {  /* Mozilla Firebox 4 to 18 */
    color: #f00
}

::-moz-placeholder { /* Mozilla Firefox 19+ */
    color: #f00;
}

input:-ms-input-placeholder,
textarea:-ms-input-placeholder {
    color: #f00;
}

input::-webkit-input-placeholder,
textarea::-webkit-input-placeholder {
    color: #f00;
}

React系列之一：初识React

前沿

  2015年是React的兴起的初期，2016年则React的辉煌阶段，在github的star数已经超过了Angular，在前端已然是半边天的角色。对于前端工程师而言，不乱在实际项目生产中使用还是作为理论研究都具有非常重要的意义。
  React是一个用于构建用户界面的Javascript库，和庞大全面的Angular相比，React只专注于架构中的V，即视图。React具有三大制胜法宝，在设计的时候从性能和编写等方面作出了详细的考虑，将这些总结成React的特性，分别为：

- JSX语法
- 虚拟DOM
- 组件化

为什么操作DOM很慢

  React最出彩的地方就属性能了，因为它尽量让我们避免直接操作DOM，大家都知道在访问和操作DOM的时候是很费时的，代价相对来说比较大，那为什么这么说呢？

不仅要避免操作DOM，还要减少访问DOM的次数

把DOM和JavaScript各自想象成一个岛屿，它们之间用收费桥梁连接。
                                  –《高性能JavaScript》

  
  在浏览器中，DOM和JS的实现，用的并不是同一个“东西”。在Chrome中，JS引擎是V8，而DOM和渲染则是靠WebCore库，DOM和JS是两个独立的个体。而且js执行效率相对来说比较低，不想java和c++边编译边执行。
